## 2. 四维度尽调框架

### 🧭 Business DD · 业务尽调
回答一个问题：**"这家公司真的在做它说的事情、而且做得还不错吗？"**

核心检查点：
- 产品 **亲身** 走一遍核心流程（包括注册、付费、客服、退款）。
- 至少访谈 **3-5 位真实用户**，非项目方推荐的那种。
- 复核关键指标（DAU/MAU/NDR/GMV/ARR）— 找项目方要后台截图 + 第三方交叉验证（应用商店榜单 / SimilarWeb / 数据平台）。
- 前 5 大客户集中度：>50% 标黄，>70% 标红。
- 获客漏斗从曝光到付费每一层的转化率。
- CAC 回收期：SaaS <24 个月健康，消费品 <12 个月健康。

### 💰 Financial DD · 财务尽调
回答：**"账面是真的吗？可持续吗？"**

- 审计报告（有最好，没有则自行核对管理报表）。
- 收入三路核对：合同金额 / 银行流水 / 账面确认。
- 毛利率 vs 同行：差异 >20% 必须解释。
- 现金 Runway = 账上现金 / 月均净烧钱速度。Runway <6 个月=危险，<3 个月=本轮到账前就可能断粮。
- 应收账款账龄 & 坏账准备。
- 关联交易：创始人 / 近亲属的关联公司，必查。
- 三年预测合理性：收入增速 vs 市场增速 vs 团队规模扩张，三者要相容。

### ⚖️ Legal DD · 法务尽调
回答：**"有没有雷？将来会不会爆？"**

- 工商信息 + 股权变更历史。
- 历轮融资协议一致性（回购条款、对赌、反稀释是否冲突）。
- 知识产权 **归属**：职务发明、开源依赖合规、代码所有权是否在公司名下。
- 未决诉讼 / 仲裁 / 行政处罚。
- 资质许可（ICP、IDC、支付牌照、医疗器械注册证等赛道专属）。
- 数据合规：个保法 / GDPR / 数据跨境。
- 劳动关系：社保缴纳是否全员足额（绕不开的合规点）。

### 👥 Team DD · 团队尽调
回答：**"这帮人能把事做成吗？如果做不成会不会跑？"**

- 创始人履历 **逐项核验**（LinkedIn / 前雇主 / 论文专利 / 媒体报道）。
- 背调：至少 **3 位** 前同事 / 前老板 / 老投资人（非项目方提供名单）。
- 核心团队完整度：CEO+CTO+COO+首席销售，缺 2 个以上标黄。
- 期权池：建议预留 10-15% 给未来 HR 扩张。
- 创始人个人风险：征信、对外投资、诉讼、有无其他全职。

## 3. Red Flag 清单（一票否决级）

| Flag | 说明 |
|---|---|
| 🚩 数据造假 | 关键指标严重不符、流水与账面差异 >10%。 |
| 🚩 创始人诚信 | 学历造假、履历注水、关键经历无法验证。 |
| 🚩 未披露诉讼 | 重大未决诉讼未主动披露。 |
| 🚩 IP 不在公司 | 核心技术 IP 不归公司所有。 |
| 🚩 关键合作不稳 | 核心供应商 / 大客户随时可断。 |
| 🚩 对赌/回购连环雷 | 前几轮回购条款叠加，本轮进去就在火药桶上。 |
| 🚩 合规断供 | 重要资质到期未续、正在被监管约谈。 |

> **处理原则**：Red Flag 一旦触发，先停止推进，升级到合伙人层讨论；是否继续需书面决定。

## 4. 数据室（VDR）索取清单

- 公司设立与变更文件
- 历轮融资协议 & SPA/SHA
- 财务：3 年审计 + 12 月管报 + 预测
- 合同：Top 10 客户 / Top 5 供应商 / 关键合作
- 劳动：劳动合同模板 / 社保记录 / 期权池协议
- IP：专利清单 / 商标清单 / 软著清单 / 关键代码仓库访问权
- 资质：营业执照 / 行业许可 / 合规证书
- 诉讼：历史诉讼清单 + 裁判文书

---

## 5. 尽调纪要写作规则

- 每项尽调结论 **3-5 句**，不堆材料。
- 结构：**做了什么 · 看到了什么 · 判断是什么**。
- 有疑点的必须列"下一步动作"。
