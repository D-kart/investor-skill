# 技能 04 · 竞争格局分析与市场定位

> **何时加载我**：用户提到"竞争格局 / 市场地图 / positioning / 定位图 / 对标 / 护城河 / 差异化"时。

---

## 1. 产出物
- 《竞争格局地图》（头部 / 挑战者 / 长尾）
- 《定位图》（两轴象限）
- 《护城河评分卡》（5 维度 × 1-5 分）
- 《差异化白板》（一页纸）

---

## 2. 竞争格局三层扫描

### Layer 1 · 直接竞争者
同赛道、同客群、同解法。列出前 10。

### Layer 2 · 间接竞争者
不同赛道，但解决 **同一个痛点**。
> 例：外卖的间接对手不是另一个外卖，是"在家做饭"和"便利店"。

### Layer 3 · 替代技术 / 业态
5 年维度可能颠覆现有解决方案。
> 例：Uber 的 L3 替代技术是自动驾驶。

---

## 3. 定位图（Positioning Map）

### 常用两轴组合
| X 轴 | Y 轴 | 适用场景 |
|---|---|---|
| 价格：低 ↔ 高 | 质量：低 ↔ 高 | 消费品 |
| 通用 ↔ 垂直 | 轻 ↔ 重 | SaaS |
| DIY ↔ 全托管 | 便宜 ↔ 贵 | 服务 |
| 开源 ↔ 闭源 | 自建 ↔ 托管 | 基础软件 |
| B 端 ↔ C 端 | 工具 ↔ 内容 | 互联网 |

### 做图原则
- **两轴必须是用户决策时的真实变量**，不是内部视角。
- 同一家公司不同产品线 **可以出现在不同象限**。
- 象限中应有 **空白区域**（如果没有，说明轴没选对）。
- 目标项目应 **落在空白区或挑战者区**，而不是和头部挤在一起。

## 4. 护城河定性论证（不打分）

> 专业研报不靠"评分卡"——靠**事实陈述 + 行业对比**。下面五个维度逐个论证，每个维度回答三个问题：**事实是什么 / 复制难度多大 / 主要威胁来自谁**。

### 五维度论证

| 维度 | 关键问法 | 论证证据要求 |
|---|---|---|
| **网络效应** | 多一个用户/客户/开发者，是否让其他用户的价值上升？ | 给出用户规模 vs 价值斜率的具体数据；无网络效应时直接说明 |
| **规模经济** | 规模越大单位成本越低？非线性下降还是线性下降？ | 给出当前规模下的单位成本 + 同行 / 历史对比 |
| **品牌心智** | 用户默认想到的品类选项？行业内是否有公认地位？ | 给出 NPS / 第三方排名 / 媒体声量 等可验证数据 |
| **技术壁垒** | 专利 / 工程化 / 数据组合是否可被开源/对手复制？ | 给出复制时间窗口估计（6-12-24 月）+ 复制难度 |
| **转换成本** | 客户切换到对手需付出多大代价？ | 给出迁移工时 / 数据搬运成本 / 业务停摆风险 等估算 |

### 论证输出范式

每条护城河按以下结构叙事：

```
【护城河名称】
事实证据：______________________________________________
（具体数据 / 客户案例 / 性能差距 / 出货量）

复制难度：6 个月 / 12 个月 / 24 个月 / 不可复制
（说明为什么是这个时间窗口）

主要威胁来源：__________________________________________
（具体对手 + 对方需要做什么才能追平）
```

### 章末总结
用 1-2 段叙事文字总结：
- 本公司**主要护城河来自哪 1-2 项**（如"数据飞轮 + 客户嵌入工作流"）
- **最弱的维度是什么**（如"网络效应几乎为零"）
- **未来 18 个月最大的威胁场景**（如"巨头通过收购同类公司直接进场"）

**禁止输出**：
- "总分 X / 25"
- "★★★☆☆"
- "中等偏强 / 弱护城河"等聚合性主观判断（如必须给定性结论，需挂事实论证而非简单定级）

## 6. 分析常见错误

- ❌ 把"功能差异"当"客户价值差异"。功能不同不等于有意义。
- ❌ 把"我们团队强"当护城河。团队是人，是变量，不是壁垒。
- ❌ 把"先发优势"当护城河。先发只在有网络效应时才成壁垒。
- ❌ 低估间接竞争者。
- ❌ 忽视"不作为"这个对手（用户完全可以不解决这个问题）。
