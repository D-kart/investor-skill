## 2. 价值主张 · 四要素公式

> **For {客群}，who {场景/痛点}，our {产品} is {品类} that {核心价值}。
> Unlike {主要替代方案}, our product {10x 差异}。**

每次写产品描述，都强制套这个模板。它能逼出四个关键信息。

### 举例
> For 中小 SaaS 公司的产品经理，who 需要在上线前快速验证功能假设，
> our SkipTest 是一个 AI 驱动的用户旅程模拟器，that 把传统需要 2 周的 A/B 测试压缩到 2 小时。
> Unlike 传统 A/B 平台，our product 不需要真实用户流量，置信区间由 LLM 人群模型生成。

## 3. 产品评估五步法

### Step 1 · 亲身使用
- 不看演示，自己注册、上手 30 分钟以上。
- 记录**第一次失败的点**在哪里。
- 观察**第一次哇塞的点**在哪里。

### Step 2 · 核心价值验证
- 它是 **Painkiller** 还是 **Vitamin**？
  - Painkiller：止痛药，用户会主动找、愿意付费。
  - Vitamin：维生素，用户觉得好但不急。
- **10x 法则**：和替代方案比，至少在 1 个维度（价格 / 性能 / 时间）上好 10 倍，否则很难破局。

### Step 3 · 差异化提炼
- 列出 5-10 个产品特性。
- 打标：✅ 客户能感知 · ⭕ 行业标配 · ❌ 技术自嗨。
- 保留 ✅ 前 3 条作为核心差异化。

### Step 4 · 客户证词
- 3-5 位真实付费客户的访谈。
- 问题聚焦：
  - 没用这个产品前你怎么解决这个问题？
  - 现在用它节省了什么（时间 / 钱 / 精力）？
  - 如果明天它关了你怎么办？

### Step 5 · 对照竞品
- 用同样的典型任务在 **2-3 款竞品** 上跑一遍，记录差异。

## 4. 技术壁垒定性论证（不打分）

> **重要**：本框架**不再使用 1-5 星 / 16/25 分** 这类量化评分——专业研报靠**事实陈述 + 定量数据 + 行业对比**说话，而不是主观打分。

### 评估维度

| 维度 | 关键问法 | 论证证据要求 |
|---|---|---|
| **专利与 IP** | 核心 IP 是否在公司名下？是否壁垒型？ | 列出核心专利清单、申请时间、地域覆盖、诉讼胜诉记录 |
| **核心算法 / 模型** | 能否被开源 70%+ 替代？ | 给出与开源方案的定量性能差距 / 训练数据规模 / 模型架构差异 |
| **工程化能力** | 性能 / 稳定性 / 规模是否领先？ | 大规模部署案例（出货量 / QPS / P99 延迟 / 系统可用性 SLA） |
| **数据 / 数据飞轮** | 独占数据 + 反馈闭环？ | 数据来源说明 + 增长曲线 + 飞轮闭环验证 |
| **网络效应 / 转换成本** | 多一个用户是否让每个用户更好？切换代价多大？ | 用户规模 vs 价值斜率 / 客户切换成本量化 / 嵌入工作流深度 |

### 论证输出（叙事化，非打分化）

每条壁垒按下列结构输出：

```
【壁垒名称】
事实证据：___________________________________________________
（具体专利号 / 数据规模 / 客户数量 / 性能差距 等定量陈述）

复制难度：6 个月 / 12 个月 / 24 个月 / 不可复制
（说明为什么是这个时间窗口）

5 年可持续性：基于 X / Y / Z 三个外部条件判断
（避免空喊"强中弱"）

主要威胁来源：___________________________________________
（具体竞争方 + 对方需要做什么才能追平）
```

### 总结段（章末）
用 1-2 段叙事文字总结：本公司的核心壁垒**主要来自哪 1-2 项**，**最弱的维度是什么**，**未来 18 个月最大的威胁场景是什么**。
**禁止**输出"总分 X/25 · 中等偏强"这类聚合分数。

## 5. 技术路线图审视

- 未来 6 / 12 / 18 个月 3 个里程碑。
- 每个里程碑必须具备：
  - 可衡量的指标（性能 / 规模 / 准确率）
  - 所需投入（人力 / 算力 / 数据）
  - 依赖前置条件
- 警惕"画饼式路线图"：里程碑无指标 / 无依赖 / 无资源测算。

---

## 6. 常见伪壁垒 & 伪差异

- 🚫 **"我们的 UI 更好"** —— UI 可以复制。
- 🚫 **"我们用了大模型"** —— 调 API 不是壁垒。
- 🚫 **"我们的团队更努力"** —— 这是人事风险，不是壁垒。
- 🚫 **"我们有先发优势"** —— 仅在有网络效应时成立。
- 🚫 **"我们速度更快"** —— 除非有底层架构优势，否则易被追平。
